Arquitectura - Hydra

Diseñada para ser rápida, modular y altamente paralelizable.


Motor principal

El núcleo de Hydra coordina toda la ejecución de la herramienta. Se encarga de cargar la configuración, interpretar los parámetros introducidos por el usuario, inicializar los módulos necesarios y gestionar el ciclo completo de la auditoría.


Módulos por protocolo

Hydra utiliza una arquitectura modular donde cada protocolo dispone de un módulo independiente encargado de comunicarse con el servicio correspondiente.


Servicios compatibles

Ver protocolos compatibles

SSH (Secure Shell)

Protocolo utilizado para la administración remota segura de servidores y sistemas Linux/Unix mediante conexiones cifradas.


FTP (File Transfer Protocol)

Servicio destinado a la transferencia de archivos entre equipos a través de una red. En su versión clásica no cifra la información transmitida.


HTTP Basic / HTTP Form

Mecanismos de autenticación utilizados por aplicaciones y paneles web para controlar el acceso mediante usuario y contraseña.


HTTPS

Versión segura de HTTP que cifra las comunicaciones mediante TLS.


SMTP

Protocolo encargado del envío de correo electrónico.


POP3

Protocolo utilizado para descargar mensajes de correo desde un servidor.


IMAP

Permite mantener el correo sincronizado entre varios dispositivos.


SMB

Protocolo utilizado principalmente para compartir archivos y recursos en sistemas Windows.


MySQL

Sistema gestor de bases de datos relacional muy utilizado por aplicaciones web.


PostgreSQL

Base de datos relacional de código abierto orientada a aplicaciones empresariales.


RDP

Protocolo de Microsoft para el acceso remoto mediante escritorio.


VNC

Sistema de escritorio remoto multiplataforma.


LDAP

Protocolo utilizado para consultar y administrar servicios de directorio como Active Directory.


Telnet

Antiguo protocolo de administración remota sin cifrado, actualmente reemplazado por SSH.




Esta separación facilita añadir nuevos protocolos sin modificar el funcionamiento del núcleo principal.


Gestión multihilo

Hydra distribuye el trabajo entre múltiples hilos de ejecución, permitiendo realizar numerosos intentos de autenticación de forma simultánea.

  • Mayor velocidad de ejecución.
  • Aprovechamiento de procesadores multinúcleo.
  • Configuración del número de hilos según el entorno.

Entrada de datos

Antes de comenzar la auditoría, Hydra carga toda la información necesaria para generar las combinaciones de autenticación.

  • Lista de usuarios.
  • Lista de contraseñas (wordlists).
  • Credenciales individuales.
  • Parámetros específicos del protocolo seleccionado.

Proceso de autenticación

Cada hilo envía las credenciales al servicio remoto utilizando el módulo correspondiente. El proceso se repite automáticamente hasta completar todas las combinaciones generadas.


Análisis de respuestas

Hydra analiza la respuesta devuelta por el servicio para determinar el resultado de cada intento.

  • Autenticación correcta.
  • Credenciales incorrectas.
  • Cuenta bloqueada.
  • Error de comunicación.
  • Servicio no disponible.

Salida de resultados

Una vez finalizada la auditoría, Hydra presenta un resumen con las credenciales válidas encontradas, estadísticas de ejecución y mensajes de error producidos durante el proceso.
