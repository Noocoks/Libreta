Inicialización - MailHog

mailhog docker smtp correo contenedor

Despliegue y gestión de MailHog mediante Docker para entornos de pruebas y desarrollo.


Gestión básica

Iniciar contenedor


  └─$ docker start mailhog

Detener contenedor


  └─$ docker stop mailhog

Creación del contenedor


  └─$ docker run -d \
  --restart unless-stopped \
  --name mailhog \
  -p 1025:1025 \
  -p 8025:8025 \
  mailhog/mailhog

Comportamiento del reinicio automático

  • Reinicias la VM → ✅ Sí
  • Reinicias Docker → ✅ Sí
  • Reinicias el sistema → ✅ Sí
  • El contenedor falla (crash) → ✅ Sí
  • Lo vuelves a iniciar manualmente → ✅ Sí
  • Lo detienes manualmente con docker stop → ❌ No

Comando antiguo

Método utilizado anteriormente para crear un contenedor temporal: 


  └─$ sudo systemctl start docker && \
  sudo systemctl enable docker && \
  docker run -d -p 1025:1025 -p 8025:8025 mailhog/mailhog
