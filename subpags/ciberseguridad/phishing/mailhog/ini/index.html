Inicialización - MailHog

<> Bash

Se usa con └─$ docker start mailhog

Se para con └─$ docker stop mailhog

Comando usado para crear este flujo:

└─$ docker run -d \

--restart unless-stopped \

--name mailhog \

-p 1025:1025 \

-p 8025:8025 \

mailhog/mailhog

Con este flujo se genera este container:

| Situación | ¿Se reinicia solo? |

| ------------------------------------------------------ | ------------------ |

| Reinicias la VM | ✅ Sí |

| Reinicias Docker | ✅ Sí |

| Reinicias el sistema | ✅ Sí |

| El contenedor crashea | ✅ Sí |

| Lo vuelves a iniciar manualmente (`docker start`) | ✅ Sí |

| Lo paras manualmente (`docker stop`) | ❌ No |

| ------------------------------------------------------ | ------------------ |

Comando usado por terminal para abrir un contenedor nuevo con mailhog:

└─$ sudo systemctl start docker && sudo systemctl enable docker && docker run -d -p 1025:1025 -p 8025:8025 mailhog/mailhog
