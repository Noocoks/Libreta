+ Git

Restricciones de Github.

Aquí crearemos el archivo .gitignore.

.gitignore sirve para decirle a Git qué archivos NO debe subir al repositorio.

Esto es importante porque tu .env contiene claves privadas (OPENAI_API_KEY, ANTHROPIC_API_KEY).

sin .gitignore, podrías subir tus API keys a GitHub.

Qué hace .gitignore

Cuando lo creas y escribes:

.env Claves privadas

venv/ Entorno virtual

__pycache__/ Caché de Python

*.pyc Archivos compilados

Si haces:

git status

SIN .gitignore, verías:

.env

venv/

test_openai.py

Git intentará subir TODO.

Problema grave

El venv/ puede pesar cientos de MB.

Y peor:

OPENAI_API_KEY=sk-xxxxx

quedaría público si el repo es público.

Bots escanean GitHub buscando claves filtradas.

Después de crear .gitignore:

git status

verás solo:

test_openai.py

.gitignore

Git ignorará:

.env

venv

cachés

archivos compliados

*Cómo crear el repositorio correctamente*

  1. Inicializa Git

git init

  1. Crea .gitignore en la raíz del proyecto

nano .gitignore

Contenido:

.env

venv/

__pycache__/

*.pyc

  1. Verifica

git status

Error común

Si subiste .env ANTES del .gitignore, Git seguirá rastreándolo.

Debes quitarlo manualmente:

git rm --cached .env

Luego:

git commit -m “remove env”

Regla importante en ciberseguridad, nunca subas:

.env

API keys

tokens

cookies

SSH keys

venv/

* Por eso. gitignore es obligatorio prácticamente en cualquier proyecto Python moderno.*
