IA + Git

Configuración recomendada para utilizar Git junto a proyectos de Inteligencia Artificial sin exponer claves privadas ni archivos sensibles.


¿Por qué usar .gitignore?

El archivo .gitignore permite indicar a Git qué archivos o directorios no deben subirse al repositorio.

Esto es especialmente importante cuando trabajamos con:

  • Claves API.
  • Entornos virtuales.
  • Archivos temporales.
  • Cachés de Python.

Problema habitual

El archivo .env suele contener datos sensibles:

  • OPENAI_API_KEY
  • ANTHROPIC_API_KEY
  • Tokens privados
  • Credenciales de acceso

Si estos archivos llegan a GitHub podrían quedar expuestos.

Existen bots que monitorizan repositorios públicos buscando claves filtradas.


Contenido recomendado


  .env
  venv/
  __pycache__/
  *.pyc

Con esta configuración Git ignorará:

  • Variables de entorno.
  • Entornos virtuales.
  • Cachés de Python.
  • Archivos compilados.

Inicialización correcta

1. Inicializar repositorio


 └─$ git init

2. Crear .gitignore


 └─$ nano .gitignore

3. Añadir contenido


  .env
  venv/
  __pycache__/
  *.pyc

4. Verificar


 └─$ git status

Error común

Si el archivo .env fue añadido al repositorio antes de crear el .gitignore, Git seguirá rastreándolo.

Para eliminarlo del control de versiones: 


 └─$ git rm --cached .env

 └─$ git commit -m "remove env"

No subir nunca

  • .env
  • API Keys
  • Tokens
  • Cookies
  • Claves SSH
  • venv/

En proyectos Python modernos, el uso de .gitignore es una práctica esencial.
